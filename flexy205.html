IOT網關，用於收集和提供對工業數據的安全訪問 我們的解決方案專注於希望保持競爭力的機械製造商。為了保持競爭力，我們的機器製造商著迷於優化其運營的各個方面。我們的數據IIoT網關Flexy與第一個安全的工業雲Talk2M的結合，使我們的客戶可以連接到他們的機器並從任何部署地點收集數據。

•

不斷發展您的機器功能：機器數據收集可以成為產品改進的門戶。通過跟踪關鍵警報和前面的標籤值，您可以更好地調整機器以適應將來的發行版。此外，您可以通過提供OPC UA服務器 和雲連接器來滿足您所有客戶的需求，從而擴展機器的價值。

•

增加其他收入來源：數據收集還可以為您的客戶帶來主動服務收入。簡單的警報應用程序可以減少您的響應時間並滿足服務水平協議。您甚至可以自動執行備件訂購通知，並提高機器的售後價值。

•

獲得競爭優勢：將數據轉換為可操作的信息使您可以監視和改善總體設備效率（OEE）。這些關鍵參數可以在簡單的端到端儀表板解決方案中進行監控，以供您或您的客戶隨時隨地查看。這僅僅是開始。隨著客戶的成熟，您可以根據他們的要求提供更好的本地儀表板或云儀表板。

什麼是 OPC UA （統一架構）？

它是一種用於在工業自動化應用中安全可靠地交換數據的通信協議。OPC UA 具有與平台無關的巨大優勢。它確保了來自多個供應商的設備之間的信息無縫流動。從傳感器到雲，OPC UA 都是完全可擴展的，面向對象的，並允許使用結構或模型，從而使治理和維護變得更加容易。






靈活-OPC UA客戶端

•

充分利用Flexy提供的靈活和大型協議兼容性

現在，Flexy集成了OPC UA客戶端，該客戶端與已嵌入其中的主要自動化協議一起，可實現更大的互操作性。OPC UA 的最高安全標準已在我們的客戶端中完全實現，並提供了各種級別的身份驗證和加密機制，以適合啟用了 OPC UA 服務器的 PLC / HMI 提供的安全選項。

通過利用易於配置的OPC UA客戶端來確保IT安全性要求。


靈活-OPC UA服務器

•

機器與工廠企業系統之間的安全通信通道（SCADA，MES，ERP）

借助嵌入式 OPC UA 服務器，相同的Flexy提供了簡單而安全的通信通道，可將信息從設備移動到應用程序層。 Flexy OPC UA 服務器提供了用於身份驗證和加密的安全選項， 可以在網關內部輕鬆進行管理，並符合您的 OPC UA 客戶端安全性。

SCADA應用程序的夢想成真。Flexy OPC UA 服務器是 SCADA 世界（基礎設施，水，能源應用等）的理想伴侶，此外，當您需要在工廠共享設備的關鍵價值時，它也非常有用。快速配置，完全獨立於 PLC。
• 易於集成到SCADA系統
• 無需購買和配置OPC UA服務器
• 與 OPC UA 相比 Modbus TCP 易於設置的優勢
• 利用機器數據做出更好的業務決策






安全的訪問

• 允許服務工程師無需現場就可以對設備進行故障排除和調試。                            
• 使用以太網（內置），Wifi或蜂窩（擴展卡）的安全 VPN 遠程連接。




數據採集

• 支持所有主要的 PLC 協議（西門子，羅克韋爾，三菱，施耐德等許多協議+Modbus，OPC UA，BACNET / IP）。
• 啟用警報，計算機 KPI，歷史記錄。




數據記錄和檢索

• 記錄多達一百萬個帶時間戳的數據值。
• 使用 HTTP，MQTT 或 FTP 可以輕鬆檢索或發送已記錄的文件，以進行數據聚集和分析。




電子郵件和短信通知

• 配置關鍵參數告警。
• 使用 Talk2M 服務通過電子郵件或 SMS 發送通知。                                         




本地Web Server儀表板整合

• 可使用免費 Web 開發工具 ViewON 嵌入 Web 儀表板（HTML5）以顯示實時機器數據。或者是用自定義 HMTL 語法開發部屬。
• 使用M2Web服務從智能手機或平板電腦監控這些儀表板




可使用基本腳本或 Java 編程

• 使用基本腳本自定義您的網關。
• 開發Java應用程序以使用高級功能，例如添加自定義I / O驅動程序，優化應用程序或與第三方應用程序集成。
