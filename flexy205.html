什麼是 OPC UA （統一架構）？

它是一種用於在工業自動化應用中安全可靠地交換數據的通信協議。OPC UA 具有與平台無關的巨大優勢。它確保了來自多個供應商的設備之間的信息無縫流動。從傳感器到雲，OPC UA 都是完全可擴展的，面向對象的，並允許使用結構或模型，從而使治理和維護變得更加容易。

SCADA應用程序的夢想成真。Flexy OPC UA 服務器是 SCADA 世界（基礎設施，水，能源應用等）的理想伴侶，此外，當您需要在工廠共享設備的關鍵價值時，它也非常有用。快速配置，完全獨立於 PLC。
• 易於集成到SCADA系統
• 無需購買和配置OPC UA服務器
• 與 OPC UA 相比 Modbus TCP 易於設置的優勢
• 利用機器數據做出更好的業務決策






安全的訪問

• 允許服務工程師無需現場就可以對設備進行故障排除和調試。                            
• 使用以太網（內置），Wifi或蜂窩（擴展卡）的安全 VPN 遠程連接。




數據採集

• 支持所有主要的 PLC 協議（西門子，羅克韋爾，三菱，施耐德等許多協議+Modbus，OPC UA，BACNET / IP）。
• 啟用警報，計算機 KPI，歷史記錄。




數據記錄和檢索

• 記錄多達一百萬個帶時間戳的數據值。
• 使用 HTTP，MQTT 或 FTP 可以輕鬆檢索或發送已記錄的文件，以進行數據聚集和分析。




電子郵件和短信通知

• 配置關鍵參數告警。
• 使用 Talk2M 服務通過電子郵件或 SMS 發送通知。                                         




本地Web Server儀表板整合

• 可使用 Web 開發工具 ViewON 嵌入 Web 儀表板（HTML5）以顯示實時機器數據。或者是用自定義 HMTL 語法開發部屬。
• 使用M2Web服務從智能手機或平板電腦監控這些儀表板




可使用基本腳本或 Java 編程

• 使用基本腳本自定義您的 Gateway。
• 開發Java應用程序以使用高級功能，例如添加自定義I / O驅動程序，優化應用程序或與第三方應用程序集成。




OPC UA

• 嵌入式，可在工廠內提供易於部署的數據集成。                                                                     




模塊化

• 2個插槽可插入2組擴展卡，並受益於廣泛的靈活連接。
• 擴展卡可支援 上網方式 Wi-Fi 、3G 、4G 、IOT 數據採集 RS-232/422/485 串列通訊卡、MPI ,通訊卡、DIO,AI 數位類比採集卡….等。
